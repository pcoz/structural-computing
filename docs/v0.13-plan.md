## 1. Headline goal

Close the last major carried-forward item from the v0.6-era
follow-on list: the **CP-SAT diagnostic layer**. The engine-level
primitives have been in `holant_tools` (`diagnose_constraints`,
`rewrite_constraints`, `rewrite_cpsat_model`,
`verify_cpsat_rewrite_equivalence`); v0.13 wires them through
the structural-computing orchestrator + `StructuralComputer`
wrapper.

After v0.13, users can hand their `cp_model.CpModel` instances
to the framework via a one-liner and get back either a faster
rewritten model OR an explicit "can't help here" signal.

## 2. Deliverable detail

### Deliverable 1 — Leaf evaluators (orchestrator-side)

Four new leaf evaluators in `orchestrator.py`:

- **`_diagnose_constraints_leaf`** — runs
  `holant_tools.diagnose_constraints` on a list of abstract
  `ConstraintSpec` inputs. Returns the `EncodingDiagnostic`
  dataclass.
- **`_rewrite_constraints_leaf`** — produces a
  `RewriteSetBlueprint` for the abstract constraint list.
- **`_rewrite_cpsat_model_leaf`** — operates directly on
  `cp_model.CpModel`. Returns the `CPSATRewriteResult` dataclass
  carrying `helped: bool` and `help_reason_text`.
- **`_verify_cpsat_rewrite_equivalence_leaf`** — verifies that a
  rewrite preserves the feasible set + optional objective.

All four registered under `(T2, ...)` and `(T4, ...)` in
`DEFAULT_LEAF_REGISTRY`.

**Classifier extension.** `_classify_problem` now recognises
three additional CP-SAT problem-dict shapes:

- `{"constraints": [...]}` (abstract list)
- `{"model": cp_model.CpModel, ...}` (direct rewrite)
- `{"original_model": ..., "rewrite_result": ...}` (verification)

All route to T2 with `problem_kind = "cpsat_diagnostic"`.

### Deliverable 2 — StructuralComputer wrapper methods

Four new methods on `StructuralComputer`:

- `sc.diagnose_constraints(constraints) -> EncodingDiagnostic`
- `sc.rewrite_constraints(constraints) -> RewriteSetBlueprint`
- `sc.rewrite_cpsat_model(model, *, rewrite_kinds=None) -> CPSATRewriteResult`
- `sc.verify_cpsat_rewrite(original_model, rewrite_result, *,
    enumeration_limit=10000, check_objective=True,
    max_witnesses=5) -> CPSATVerificationResult`

All delegate through `self._delegate(...)` (per the v0.12
consolidation pattern). Three new pytest tests in
`tests/test_smoke.py` (gated on `pytest.importorskip("ortools.sat.python.cp_model")`).

## 3. Worked example

```python
from structural_computing import StructuralComputer
from ortools.sat.python import cp_model

sc = StructuralComputer()
model = cp_model.CpModel()
xs = [model.NewBoolVar(f"x{i}") for i in range(4)]
model.Add(sum(xs) == 2)

# Pre-flight the model: is there a holographic shortcut?
result = sc.rewrite_cpsat_model(model)
if result.helped:
    print("Got a structurally cheaper rewrite:")
    print(" ", result.help_reason_text)
    # Solve the rewritten model with the regular CP-SAT solver:
    solver = cp_model.CpSolver()
    solver.Solve(result.rewritten_model)
else:
    print("No matchgate shortcut here; running original.")
    solver = cp_model.CpSolver()
    solver.Solve(model)

# Optional: verify equivalence on small instances.
verify = sc.verify_cpsat_rewrite(model, result, enumeration_limit=1000)
assert verify.equivalent
```
