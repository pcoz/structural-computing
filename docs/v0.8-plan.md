### Deliverable 1 — Higher-m augmented Plücker (holant-tools v0.7.0)

**Engine side (`holant-tools/non_symmetric.py`):**

`matchgate_identities_arity_n_odd_augmented(tau, arity)` was
shipped in v0.6.0 with hardcoded m=1 case and extended in v0.6.1
with hardcoded m=3. v0.8 D1 (holant-tools v0.7.0) generalises to
enumerate every viable m via a single loop:

```python
m = 1
while m + 5 <= n:
    if max_m is not None and m > max_m:
        break
    for M_subset in combinations(positions, m):
        S_base = list(M_subset) + [omega_idx]
        remaining = [i for i in positions if i not in M_subset]
        for abcd in combinations(remaining, 4):
            identities.append(expand(_plucker_identity(S_base, abcd)))
    m += 2
```

**New kwarg `max_m`** (keyword-only):
- `max_m=None` (default) — enumerate every viable m. The v0.7.0
  behaviour.
- `max_m=1` — reproduces v0.6.0 (m=1 only).
- `max_m=3` — reproduces v0.6.1 (m ∈ {1, 3}).
- `max_m=5` — caps at m=5 (useful at high arity to bound work).

**Identity counts (cumulative across m):**

| arity | m=1 | m=3 | m=5 | m=7 | total |
|---|---|---|---|---|---|
| 6 | 30 | (n/a) | (n/a) | (n/a) | 30 |
| 8 | 280 | 280 | (n/a) | (n/a) | 560 |

**Symbolic + numerical verification:**
- m=5 case verified symbolically on a generic 11x11 skew matrix:
  the Plücker quadratic vanishes identically.
- Numerical verification on signatures built from random
  (n+1)x(n+1) skew matrices: all 6720 identities at arity 10
  vanish to ~1e-14.

**Behaviour-change note for upgraders:** the default at arity ≥
10 changed (more identities returned). This is strictly
strengthening: realisable signatures still pass; non-realisable
signatures may be caught faster. To preserve v0.6.1 behaviour
exactly, pass `max_m=3`.

### Deliverable 2 — Fundamental-cycle backup (4th tier of LT cascade)

**File: `structural-computing/structural_computing/decompose.py`.**

The Lipton-Tarjan cascade now has four tiers:

1. **v0.4** — simple BFS-layer case (textbook LT).
2. **v0.5** — spanning-tree tree-edge backup (balanced subtree
   split + cross-edge augmentation).
3. **v0.6 D2** — level-based + articulation-augmentation backup
   (BFS-level + bin-packed components + highest-degree
   articulation vertex).
4. **v0.7 D2** (this release, mid-arc) — fundamental-cycle backup.

**Algorithm of the fourth tier:**

1. Reconstruct the BFS spanning tree T from the BFS levels.
2. Enumerate non-tree edges (m − n + 1 of them in a connected
   graph).
3. For each non-tree edge e = (u, v):
   a. Compute the fundamental cycle C_e = path_T(u → v) ∪ {e}.
      Always a SIMPLE cycle (T is a tree, path is unique).
   b. Remove C_e vertices from the graph; compute residual
      connected components.
   c. Jordan-curve theorem: in a planar graph, at most TWO main
      components emerge (the "inside" and "outside" of C_e in
      the planar embedding).
   d. If |C_e| ≤ 2·sqrt(2n) AND max(component) ≤ 2n/3, this
      cycle is a valid separator candidate.
4. Pick the candidate with the best balance score
   (smaller |S|, more balanced (|A|, |B|)).

**Helper `_fundamental_cycle(tree_parent, u, v)`** computes the
cycle via lowest-common-ancestor walk.

**Honest scope:** Works WITHOUT requiring rotation system input.
The Jordan-curve property is invoked implicitly. The full LT 1979
paper's planar-dual argument additionally PROVES that an
acceptable non-tree edge always exists when the simple case fails;
our search is bounded-effort. On adversarial graphs where no
non-tree edge produces a valid split, we raise ValueError and the
caller falls back to a user-supplied separator.

**Cascade end-to-end:** when D2 succeeds, the orchestrator's
`PlanarSeparator(auto=True)` path now uses the four-tier cascade;
the docstring updates (already shipped in v0.7 D9) reflect the
extended cascade.
