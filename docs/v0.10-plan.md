## 1. Headline goal

Wire holant-tools' polynomial-time tropical-Holant kernel into the
structural-computing orchestrator as a new leaf-evaluator family.
The engine-level tropical primitives have been in `holant-tools`
since v0.2.0a* (`semiring`, `pfaffian(M, semiring=...)`, `tropical`,
`min_cost_schedule`, etc.) but were not reachable through the
orchestrator's `evaluate()` entry point or the `StructuralComputer`
one-liner. v0.10 closes that gap for the two highest-impact
questions:

- `min_weight_matching` on weighted graphs
- `min_cost_schedule` on `SchedulingInstance` objects

The same admissible-set machinery that counts perfect matchings
under the standard (+, ×) semiring now computes the **cheapest**
admissible configuration under the tropical (min, +) semiring.

## 2. Deliverable detail

### Deliverable 1 — `_min_weight_matching_leaf`

**File:** `structural-computing/structural_computing/orchestrator.py`

A new leaf evaluator dispatched on (T2, "min_weight_matching") and
(T4, "min_weight_matching"). Accepts a problem dict with
`vertices`, `edges`, and an optional `weights` dict; builds an
n × n weight matrix; calls
`holant_tools.min_weight_perfect_matching(W)` which dispatches
internally between Hungarian (O(n³), bipartite) and Edmonds blossom
via NetworkX (O(n³), general non-bipartite). Returns:

```python
{"cost": float, "matching": [(u, v), ...], "feasible": bool}
```

Infeasible inputs (odd vertex count, no perfect matching) return
`feasible=False`, `cost=matching=None`.

**Wrapper method on StructuralComputer:**

```python
def min_weight_matching(self, graph, weights=None) -> dict:
    ...
```

### Deliverable 2 — `_min_cost_schedule_leaf`

**File:** `structural-computing/structural_computing/orchestrator.py`

Leaf evaluator for `holant_tools.SchedulingInstance` inputs.
Accepts a problem dict with `instance` (the SchedulingInstance),
`cost_fn` (callable `(job, machine, slot) -> float`), and optional
per-job constraints (`allowed_machines`, `time_windows`,
`forbidden_edges`). Delegates to `holant_tools.min_cost_schedule`.

Returns:

```python
{"cost": float, "schedule": dict, "feasible": bool}
```

where `schedule` is a dict mapping job name → (machine name, time slot).

**Wrapper method on StructuralComputer:**

```python
def min_cost_schedule(self, instance, cost_fn, *,
                       allowed_machines=None,
                       time_windows=None,
                       forbidden_edges=None) -> dict:
    ...
```

## 3. Honest scope (carried forward)

v0.10 wires two questions. The engine has more tropical primitives
already shipped that the orchestrator doesn't yet dispatch to:

- `min_cost_flow` (network-flow problems)
- `min_cost_roster` (rostering / nurse-scheduling instances)
- `min_cost_dedup` (entity deduplication)
- `tropical_instance_coordinates` (one-call "is this instance
  structurally well-suited for tropical optimisation?" diagnostic)
- Polymorphic `pfaffian(M, semiring=...)` direct access
- Polymorphic `holant_sum_of_pfaffians`, `holant_genus_g`,
  `holant_planar` (the multilinear evaluators that already accept
  any semiring at the engine layer)

These will follow as v0.11+ deliverables; the engine is already
production-ready, only the orchestrator-side dispatch wiring is
missing.
