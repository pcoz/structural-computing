## 1. Why v1.0.0 now

The v0.6-era carried-forward list closed at v0.13. All five major
items are shipped:

| # | Item | Arc |
|---|---|---|
| 1 | Higher-m augmented Plücker | v0.8 ✅ |
| 2 | Full Lipton-Tarjan 1979 with planar-dual | v0.9 ✅ |
| 3 | Tropical optimisation in orchestrator | v0.10–v0.11 ✅ |
| 4 | Wrapper consolidation | v0.12 ✅ |
| 5 | CP-SAT diagnostic + rewrite layer | v0.13 ✅ |

The public API is feature-complete for the use cases the package
was designed for: exact counting, reliability, min-cost
optimisation, CP-SAT pre-flight. The v0.x line accumulated 302
tests across ~15 modules; the test surface is broad enough to
catch regressions on the stable methods.

v1.0.0 is the natural milestone: declare API stability so
downstream packages (starting with `structural-computing-bench`)
can pin `structural-computing>=1.0,<2.0` safely.

### Deliverable 2 — Calibration runs

Ran `structural-computing-bench`'s `run_calibration.py` on the
current machine (`AMD64-Windows-11`, Python 3.13.13) with
`--repeat 3`. Produced `calibration_data/default.py` containing:

- `CALIBRATED_COSTS`: dict keyed by `(tier, question)`, value =
  `{model: "power_law" | "exponential", a, b, rms}`.
- `METADATA`: machine identifier, Python version, bench version,
  generation timestamp.

Verified the data loads through
`apply_calibration(CALIBRATED_COSTS)` and `predict_seconds(tier,
question, n=...)` returns sensible wall-clock estimates.

The bench's default-evaluator menu doesn't yet include the
v0.10–v0.13 tropical + CP-SAT leaves; calibrating those is a
small bench-side follow-on for v1.1.0+.

### Deliverable 3 — API stability review

Created `docs/STABILITY.md` documenting the per-symbol stability
contract:

- **Stable tier**: `StructuralComputer` + all its methods (~30),
  `Classification` + classifier helpers, `Orchestrator` +
  `OrchestratorResult` + `WorkflowStep`, the public exceptions,
  the calibration helpers.
- **Experimental tier**: reductions / compositions /
  decompositions classes, pipeline-router framework, verifier
  helpers, replay cache, trace aggregator.
- **Internal tier**: anything underscore-prefixed.

The stability contract follows standard semver post-v1.0.0:
breaking changes to Stable-tier symbols require a major version
bump (v2.0.0); minor versions (v1.1.0, v1.2.0, ...) may break
Experimental-tier; patches are bug-fix-only.

## 3. What v1.0.0 does NOT include (intentional)

- **Domain DSLs** (workflow-systems, catastrophe-modelling,
  build-system audit) — sketched in the research roadmap; not
  yet code. These are year-10 deliverables, not v1.0 polish.
- **Calibration data for tropical + CP-SAT leaves** — per-machine
  task; the bench's default-evaluator menu would need extending
  to include them, then a re-run. v1.1.0+ work.
- **Further tutorial expansion** — the README has a "more worked
  examples" section but a 30-minute end-to-end tutorial (à la
  the `free-fermion-quantum-simulation/docs/getting-started.md`)
  would be valuable for downstream adopters.

## 4. After v1.0.0

The framework now has:
- 302 passing tests
- All five v0.6-era major follow-ons shipped
- An API stability contract
- Wall-clock calibration data for the principal leaves
- Worked examples for every public method category

The user-facing capability is:

> `pip install structural-computing` → a `StructuralComputer`
> one-liner for exact counting + reliability + min-cost
> optimisation + CP-SAT pre-flight on combinatorially structured
> problems, with honest stops on out-of-family inputs.

That's the v1.0.0 deliverable. Downstream packages can build on
it now.
