## 4. Deliverable 2 — Full Lipton-Tarjan 1979 backup with planar dual

### 4.1 What ships

`_lipton_tarjan_separator` succeeds on EVERY connected planar graph
where the LT theorem applies, with the size bound `|S| ≤ 2*sqrt(2n)`
guaranteed by construction (not by post-hoc check). The v0.5 tree-
edge backup remains as a faster first-line attempt; the full LT
1979 fires only when the tree-edge backup fails to bound `|S|`.

### 4.2 The algorithm

Lipton & Tarjan 1979, *A Separator Theorem for Planar Graphs*, SIAM
J. Appl. Math. 36(2). The original proof:

1. **Pick the middle BFS level** `L*` — the level such that levels
   strictly above contain `< n/2` vertices and levels up to and
   including `L*` contain `≥ n/2`.
2. **If `|L*| ≤ 2*sqrt(2n)`:** simple case, return immediately.
3. **Else: find a balanced level pair `(L_a, L_b)` with
   `a < L* < b`** such that:
   - All levels strictly above `L_a` have `< n/3` vertices.
   - All levels strictly below `L_b` have `< n/3` vertices.
   - `|L_a| ≤ 2*sqrt(2n) / 2` and `|L_b| ≤ 2*sqrt(2n) / 2`
     (these are the "skinny" levels that bracket the fat middle).
4. **Within the middle band `(L_a, L_b)`**:
   - The induced sub-graph is connected (since `L_a` is a BFS
     separator).
   - Build the **BFS spanning tree T** within the middle band.
   - Look at non-tree edges of the original graph that lie within
     this band. Each non-tree edge `(u, v)` has a fundamental cycle
     in T.
   - On a PLANAR graph, each fundamental cycle is the boundary of
     a planar face (in the embedding); the "inside" of the cycle
     contains some sub-set of the band's vertices.
   - Find a non-tree edge whose fundamental cycle has its inside
     of size `≤ 2n/3` AND its outside of size `≤ 2n/3` (within
     the band).
   - The separator is `L_a ∪ L_b ∪ (vertices on the fundamental
     cycle)`. The cycle has size `≤ 2 × height_of_band_BFS_tree`
     which for planar graphs is `O(sqrt(n))`.
5. **The bound:** `|L_a| + |L_b| + |cycle| ≤ 2*sqrt(2n)` by the
   theorem's quantitative analysis (Lipton-Tarjan §3).

### 4.3 Implementation

`structural_computing/decompose.py`:

1. New private function `_lipton_tarjan_full_backup(problem,
   levels, ...)` invoked from `_lipton_tarjan_separator` only when
   the v0.5 tree-edge backup fails.
2. Subroutine `_find_balanced_level_pair(levels, n)`: returns
   `(a, b)` per step 3 above.
3. Subroutine `_fundamental_cycle_separator(rotation, band_levels)`:
   the inside-vs-outside-of-cycle counting via the planar
   embedding. This is the algorithmically hardest part —
   requires:
   - The rotation system (rotation system of the original graph,
     restricted to the band).
   - Building the planar dual graph of the band restriction.
   - Computing fundamental cycles and their enclosed regions.

   Possible simplifications for v0.6 first cut:
   - Use a direct "tree-DFS + face-tracking" algorithm rather than
     building the full planar dual.
   - Accept slightly weaker bounds (e.g. `|cycle| ≤ 3 × height`)
     if it makes the implementation tractable.

4. **Honest-scope guard.** If the rotation system is missing or
   the band restriction produces a non-cellular embedding, raise
   `ValueError` with a clear message (caller falls back to user-
   supplied separator).

### 4.4 Acceptance criteria

- On a corpus of 5-7 "fat middle level" planar graphs that defeat
  both v0.4's simple case AND v0.5's tree-edge backup, the new
  full backup finds a valid `(S, A, B)` partition with
  `|S| ≤ 2*sqrt(2n)`.
- The corpus must include: a planar grid with a "wide row" in the
  middle (heuristically planar); a planar graph constructed from
  a degenerate genus-0 tree + a few extra edges; an adversarial
  case from the LT 1979 paper if one is mentioned.
- All 272 v0.5 tests still pass (the v0.4 simple case and v0.5
  tree-edge backup take precedence, so most cases never reach
  the full backup).
- Honest-stop on non-planar inputs (the LT bound doesn't apply).

### 4.5 Estimated effort

**5-7 days.** The hardest item in v0.6. The fundamental-cycle
inside/outside counting on a planar embedding is genuinely complex;
the corner cases (non-cellular residual embeddings, degenerate
cycles where the "inside" is empty or the entire band) need
careful handling. May need a simplification pass partway through
to keep scope manageable.

## 5. Deliverable 3 — |S| ≥ 3 augmented Plücker configurations at arity ≥ 8

### 5.1 What ships

The engine function (D1's promoted helper) extended to handle
`|S \ {omega}| ∈ {3, 5, 7, ...}` configurations at arity ≥ 8.
Together with the v0.5 `|S|=2` case, this characterises the
d-admissibility variety completely (modulo the `|S|=odd` cases
which only contribute at odd arity — separate work, deferred to
v0.7).

### 5.2 The math

Per the v0.5 plan §3.2 configuration analysis: for the Plücker
identity on the augmented `(n+1)`-vertex Kasteleyn matrix to give
a PURE-τ relation, all 8 subsets `T = S ∪ X` (with `X ⊆ {a,b,c,d}`)
must include `omega` and have even cardinality.

The viable configurations were:
- `|S \ {omega}| = 1` → `|T|` ∈ `{2, 4, 6}` (v0.5 D1 ships this).
- `|S \ {omega}| = 3` → `|T|` ∈ `{4, 6, 8}`. At arity 8: weight-5,
  weight-3, weight-1 τ values. NEW at arity 8.
- `|S \ {omega}| = 5` → `|T|` ∈ `{6, 8, 10}`. Requires arity ≥ 10.
- `|S \ {omega}| = 7` → arity ≥ 12.

For v0.6 D3, the `|S| \ {omega}| = 3` case is the next step.

### 5.3 Implementation

After D1 (helper promoted to holant-tools), extend it in the engine
to enumerate ALL valid configurations:

```python
def matchgate_identities_arity_n_odd_augmented(tau, n):
    # |S|=2 case (v0.5 D1)
    for p in range(n):
        for abcd in combinations(remaining, 4):
            ...
    # |S|=4 case (v0.6 D3): S = {omega, p, q, r} with p, q, r
    # distinct in {0..n-1}, 4-subset in remaining.
    if n >= 8:
        for pqr in combinations(range(n), 3):
            remaining = set(range(n)) - set(pqr)
            for abcd in combinations(remaining, 4):
                ...
    # |S|=6 case (v0.7+): only at n >= 10.
    return identities
```

The Plücker quadratic identity structure is the same; what changes
is the `S` base set and the mapping of Pfaffians to τ values.

### 5.4 Acceptance criteria

- At arity 8, the augmented enumeration count grows from `8 ×
  C(7, 4) = 280` (|S|=2 only, v0.5 carry-over) to
  `8 × C(7, 4) + C(8, 3) × C(5, 4) = 280 + 56 × 5 = 560` (|S|=2
  plus |S|=4).
- Symbolic verification on a generic 9x9 skew matrix shows all
  new identities vanish identically.
- Numeric verification on signatures built from random 9x9 skew
  matrices.
- The new `realisability_check` value could be
  `"plucker_arity_n_full_v06"` or similar (or we keep
  `"plucker_arity_n_full"` and let the higher-arity cases be
  silently tighter — the user-facing semantic is "the check is
  proven-sufficient as of this version").

### 5.5 Estimated effort

**3-4 days.** Easier than v0.5 D1 (which was the first augmented
case): the configuration-analysis pattern is established, the
engine-side framework exists post-D1, and the symbolic verification
methodology is also established.
